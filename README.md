# arXiv:2604.16063

This repository contains the [Julia](https://github.com/JuliaLang/julia) implementation for the numerical simulations presented in the paper **"Spinon shift current in a noncentrosymmetric quantum spin chain"** ([arXiv:2604.16063](https://arxiv.org/abs/2604.16063)). All simulations are performed using the [ITensors.jl](https://github.com/ITensor/ITensors.jl) library.

This README is partly generated by Google Gemini.

## File Overview

* **`itebd-spin_2sites.jl`**: Contains the main simulation logic, including the Vidal MPS representation, gate application functions, and high-level loops for imaginary-time (`itebd_gs_2site`) and real-time (`itebd_realtime_2site`) evolution.
* **`itebd-spin_sub.jl`**: Contains the core physics definitions, including the Hamiltonian, perturbation operators, external field functions, and measurement functions for polarization and current.

## Prerequisites

Ensure you have Julia installed along with the required packages. Note that in modern ITensors.jl, `ITensorMPS` is often needed as a separate package or comes bundled; based on your scripts, you should ensure both are available:

```julia
using Pkg; Pkg.add(["ITensors", "ITensorMPS", "HDF5"])

```

## Usage Workflow

### 1. Configuration

Before running, you must define the physical parameters as global variables in your main script or REPL. These include:

* **Hamiltonian constants**: `Jxy`, `Jz`, `h`, `b1`.
* **Polarization (Perturbation) constants**: `Pxy`, `Pz`, `b2`.
* **External electric Field**: `E_amp`, `Omega`, `Ncyc`.

### 2. Finding the Ground State

To find the ground state of the system, use the `itebd_gs_2site` function. This performs imaginary-time evolution using a 4th-order Suzuki-Trotter decomposition.

```julia
# Setup sites
sites = siteinds("S=1/2", 2)

# Parameters
beta_step_init = 0.1
maxdim = 100

# Run ground state search
itebd_gs_2site(sites, beta_step_init, maxdim)

```

**Output:** This will save the ground state MPS to `MPS-GS.h5` and the singular values ($\Lambda$) to `Lambda-GS.h5`.

### 3. Real-Time Evolution

Once the ground state is saved, you can simulate dynamics (e.g., under an external field) using `itebd_realtime_2site`.

```julia
t_step = 0.01        # Time step size
step_period = 10    # Frequency of measurements
N_t = 1000          # Total number of steps
input_name = "GS"   # Looks for MPS-GS.h5
output_name = "dyn" # Suffix for output CSV files

itebd_realtime_2site(maxdim, t_step, step_period, N_t, input_name, output_name, true)

```

## Output Files

The simulation generates several `.csv` files for time-resolved analysis:

* **`Sz_[output_name].csv`**: Magnetization $\langle S^z \rangle$ for each site.
* **`polarization_[output_name].csv`**: Effective electric polarization of the system.
* **`current_[output_name].csv`**: Calculated displacement currents. `current1` and `current2` are distinguished by their dependence on the anisotropy $\Delta \equiv J_z/J_{xy}$ (For details, please refer to the original paper).
* **`TE-Err_[output_name].csv`**: Entanglement entropy (EE), bond dimensions (dim), and truncation errors to monitor simulation convergence.
